# Security Policy

## Supported Versions

Security fixes are prioritized for the latest published release on `main`.

## Reporting a Vulnerability

Please do not open public issues for security reports.

Use GitHub Security Advisories (private reporting) for this repository.
If private reporting is unavailable, contact project maintainers directly and include:

- affected version/commit
- reproduction steps
- impact assessment
- suggested remediation (if available)

## Response Process

1. Acknowledge report receipt.
2. Validate and triage severity.
3. Prepare and test a fix.
4. Publish coordinated disclosure and patched release notes.
