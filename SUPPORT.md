# Support

## Getting Help

- Usage questions and troubleshooting: open a GitHub issue.
- Feature ideas: use the feature request issue template.
- Security concerns: follow `SECURITY.md` and use private reporting.

## Before Opening an Issue

1. Read `README.md`.
2. Check docs in `docs/`.
3. Reproduce with latest `main` branch.
4. Include logs, config context (without secrets), and exact commands.
